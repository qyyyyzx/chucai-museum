# 楚菜文化数字博物馆 - 团队分工详细指南

> 本文档面向零基础开发人员，按重要性从高到低排列，每个任务都有明确的验收标准。

---

## A. 基础设施搭建（最高优先级，所有人依赖此项）

### A1. 项目初始化负责人

**任务描述**：确保所有人都能把项目跑起来

**具体工作**：
1. 确认 `npm install` 能正常安装依赖
2. 确认 `npm run dev:h5` 能启动开发服务器
3. 确认 `npm test` 能跑通测试
4. 把遇到的问题和解决方案记录到 `docs/setup-faq.md`

**验收标准**：
- [ ] 本地能打开 http://localhost:5173 看到页面
- [ ] 运行 `npm test` 显示所有测试通过
- [ ] 写了一份新人上手指南文档

**预计时间**：1-2 天

---

### A2. 通用工具开发负责人

**任务描述**：开发所有模块都要用的基础工具

**具体工作**：
1. 阅读 `src/shared/llm.md` 了解约束
2. 实现 `src/shared/utils/route-query.js` - 路由参数处理工具
3. 实现 `src/shared/utils/image-encode.js` - 图片转 base64 工具
4. 为每个工具函数编写单元测试，放在 `tests/modules/shared/` 目录

**验收标准**：
- [ ] 每个工具函数都有对应的测试文件
- [ ] 运行 `npm test` 全部通过
- [ ] 工具函数有中文注释说明用途和参数

**预计时间**：3-5 天

---

### A3. 通用组件开发负责人

**任务描述**：开发所有页面都要用的基础 UI 组件

**具体工作**：
1. 阅读 `src/shared/llm.md` 了解约束
2. 开发以下组件（必须使用 `@dcloudio/uni-ui`）：
   - 退出/返回按钮组件（用于 reLaunch/switchTab 页面）
   - 加载状态组件
   - 错误提示组件
3. 每个组件写一个使用示例页面

**验收标准**：
- [ ] 组件在 H5 模式下能正常显示
- [ ] 没有使用任何 emoji 或装饰性符号
- [ ] 所有文本输入使用 `<uni-easyinput>` 而非原生 `<input>`

**预计时间**：3-5 天

---

## B. 用户模块（所有业务模块依赖登录态）

### B1. 用户登录功能负责人

**任务描述**：实现用户登录和登录态管理

**具体工作**：
1. 阅读 `src/modules/user/llm.md` 了解模块边界
2. 在 `src/modules/user/domain/` 实现用户数据模型
3. 在 `src/modules/user/services/` 实现登录业务逻辑
4. 在 `src/platform/mock/` 实现 H5 端的 mock 登录（用 localStorage 模拟）
5. 在 `cloudfunctions/` 实现微信端的真实登录云函数
6. 编写单元测试

**验收标准**：
- [ ] H5 模式下能模拟登录，刷新页面后登录态保持
- [ ] mock 和云函数的接口签名完全一致（方法名、参数、返回值结构相同）
- [ ] 有至少 3 个测试用例（登录成功、登录失败、获取用户信息）

**预计时间**：5-7 天

---

### B2. 用户信息页面负责人

**任务描述**：开发用户个人信息展示和编辑页面

**具体工作**：

**验收标准**：
- [ ] 页面能正常显示用户信息
- [ ] 编辑后能保存（H5 模式保存到 localStorage）
- [ ] 没有使用原生 `<input>` 或 `<textarea>`
- [ ] 没有使用任何 emoji

**预计时间**：3-5 天

## C. 数字展陈模块（核心业务功能）

### C1. 展陈数据模型负责人

**任务描述**：定义楚菜展品的数据结构

**具体工作**：

3. 编写数据模型的验证函数和测试

**验收标准**：
- [ ] 数据模型有完整的 JSDoc 注释
- [ ] 验证函数能检测必填字段缺失
- [ ] 测试覆盖正常情况和异常情况

**预计时间**：3-5 天

---

**具体工作**：

**验收标准**：
- [ ] 列表能正常显示 mock 数据
- [ ] 下拉刷新能获取新数据

**预计时间**：5-7 天

---

### C3. 展陈详情页面负责人

## 通用规范（所有人必须遵守）

### 提交代码前检查清单

1. **运行测试**：`npm test` 必须全部通过
2. **检查 emoji**：`grep -rP '[●✓✗★☆→←↑↓▲▼◆◇■□◉◎♥♡⚠⚡]' src` 必须零匹配
3. **检查原生输入框**：不能有裸的 `<input>` 或 `<textarea>`
4. **文档同步**：修改了代码必须更新对应的 `llm.md`
5. **提交信息格式**：`[type][你的名字] 具体改动内容`

### 分支规范

1. 禁止在 main 分支直接开发
2. 从 main 拉新分支：`git checkout -b feat/你的功能名`
3. 完成后通过 Merge Request 合入 main

### 遇到问题怎么办

1. 先看对应模块的 `llm.md` 文档
2. 搜索项目中的类似代码参考
3. 记录问题到 `docs/issues.md`，开会讨论
